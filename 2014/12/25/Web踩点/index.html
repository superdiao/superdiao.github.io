Web安全之踩点思路

发表于 2014-12-25   |   分类于 Web安全   |     |   阅读次数

Web踩点

一、什么是踩点

对白帽子来说，在开始真正渗透之前，必须完成渗透前的信息收集，收集的第一步就是踩点。彩电就是对指定目标进行系统的网站信息收集，攻击者将尽可能多的收集目标单位的安全情况的方方面面。最后就可以得到一个独特的足迹。

因为数据的大量泄漏加上上网的群众大都不注重自己的信息保密，导致只要想查，任何人都可以从公开渠道查到很多，其数量之多绝对会让你们大吃一惊。

总之：成功的攻击者需要这样，他们必须收集大量的信息，以便集中火力进行外科手术式攻击（这种攻击不会轻易被捉住）

二、为什么要踩点

每个人都有自己的习惯、思维方式，发掘他的习惯、思维方式，就是他的弱点。

踩点是必须的，基于一个根本的原因，它能展示给你一个黑客能看到的情景。如果你看到黑客所看到的，你就会知道你的环境中可能暴漏什么潜在的安全漏洞。当你知道了你所暴漏的信息，你就会知道如何去防范。

踩点是最辛苦的任务之一；同时，对于试图进行模拟黑客渗透尝试的初学者来说，它也是最枯燥的。但它同时也是最为重要的任务之一。决定渗透工作成败与效率的主要因素。所以踩点必须精确地进行，而且必须在受控状态下完成。

三、Web踩点有哪些常见手段

1、网站共享主机（旁注）

旁注是网络上比较流行的一种入侵方式，利用同意主机上面不同的网站漏洞得到webshell，从而利用主机上的程序或者服务所暴漏的用户所在的物理路径进行入侵。网站共享主机漏洞是更为严谨的学术叫法。

●k8_c段旁注查询工具

k8_c段旁注工具利用微软提供的API查询目标服务器或C段服务器上绑定的其它网站。

●关于c段旁注
旁注与c段嗅探的意义，旁注的意思就是从同台服务器上的其他网站入手，提权，然后把服务器端了，就自然把服务器端了。c端嗅探就是拿下同一个c段中的其中一台服务器，然后利用工具嗅探拿下该服务器。

2、WHOIS

利用whois服务，可以获取与目标相关的具体信息，包括注册人邮箱，IP地址，公司dns主机名以及地址和电话号码等敏感信息。

①whois正向查询

②whois反向查询

whois查询网站：爱站网、站长之家

3、二级域名

①搜索引擎使用site指令搜索

百度、谷歌、必应

②dns区域传送/域名暴力枚举

fierce是使用多种技术来扫描目标主机IP的一个DNS服务器枚举工具。运用递归的方式来工作。工作原理是先通过查询本地DNS服务器来查找目标DNS服务，然后使用目标DNS服务器来查找子域名。fierce的主要特点就是可以用来定位独立IP空间对应域名和主机名。

在kali上输入以下指令：
fierce -dns xxx.com -threads 100

4、CDN后的真实IP

CDN概述：

CDN即内容分发网络。其基本思路是尽可能避开互联网上有可能影响数据传输速度和稳定性的瓶颈和环节，使内容传输的更快、更稳定。通过在网络各处放置节点服务器所构成的在现有的互联网基础之上的一层只能虚拟网络，CDN系统能够实时地根据网络流量和各节点的链接、负载状况以及用户的距离和相应时间等综合信息将用户的请求重新导向离用户最近的服务器节点上。其目的是使用户就近取得所需内容，解决Internet网络拥挤的状况，提高用户访问网络的响应速度。

CDN是构建在网络之上的内容分发网络，依靠部署在各地的边缘服务器，通过中心平台的负载均衡、内容分发、调度等功能模块，使用户就近获取所需内容，降低网络拥塞，提高用户访问响应速度、CDN的关键技术是内容存储和分发技术。

phpinfo中server_addr的IP地址百分百是真实服务器的IP地址

网上搜索phpinfo的方法：intitle:phpinfo{} inurl:*.php

5、C段子网网关

通过路由跟踪排除C段不同网关邻居主机

windows下的命令：只支持icmp（ping）协议，有些服务器会丢弃icmp协议

 tracert [-d] [-h maximum_hops] [-j host-list] [-w timeout][-R] [-S srcaddr] [-4] [-6] target_name
-d                 不将地址解析成主机名。
-h maximum_hops    搜索目标的最大跃点数。
-j host-list       与主机列表一起的松散源路由(仅适用于 IPv4)。
-w timeout         等待每个回复的超时时间(以毫秒为单位)。
-R                 跟踪往返行程路径(仅适用于 IPv6)。
-S srcaddr         要使用的源地址(仅适用于 IPv6)。
-4                 强制使用 IPv4。
-6                 强制使用 IPv6。

linux下的命令：

traceroute -n -T -p 53 ip地址
traceroute -n -T -p 80 //优先
traceroute -T  -p 80 
-n 不解析主机名
-p指定端口
-T指定协议为TCP
-I指定协议为icmp
-U指定协议为udp
-S指定发送源端口
ftp://ftp.cerias.purdue.edu/pub/tools/unix/netutils/traceroute/old
增强版可指定源端口

6、IDC机房ISP信息

通过IP查询出ip段所分配运营商。

全球注册信息管理机构：
北美 arin.net
欧洲 ripe.net
亚洲www.apnic.net
拉丁美洲/加勒比whois.lacnic.net

①IP数据库管理机构查询IP信息。

②目标IP网段或者临近网段路由器开放80管理端口泄露IDC运营商信息。

nmap -Pn -n -sC -sV -p 80 211.147.4-6.1-254
-Pn 不进行主机发现
-n 不用域名解析，加快扫描速度
-sC 搜集各种应用服务的信息
-sV 探测端口的服务类型/具体版本等信息
扫描目标网段的http-title，获取路由器title，扫描的话扫此网段的周边几个网段。
